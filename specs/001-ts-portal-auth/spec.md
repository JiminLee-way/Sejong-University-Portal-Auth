# Feature Specification: Sejong Portal Auth (TypeScript)

**Feature Branch**: `001-ts-portal-auth`
**Created**: 2026-03-19
**Status**: Draft
**Input**: TypeScript로 세종대학교 포털 SSO 인증 및 데이터 조회 시스템 구축

## User Scenarios & Testing

### User Story 1 - 기이수 성적 조회 (Priority: P1)

개발자가 학생의 세종대 포털 계정(학번/비밀번호)을 사용하여 해당 학생의 전체 기이수 성적 데이터를 프로그래밍 방식으로 가져온다. 성적 데이터에는 과목명, 학점, 등급, 평점, 이수구분 등 전체 성적 정보와 GPA, 취득학점 등 집계 정보가 포함된다.

**Why this priority**: 성적 조회가 가장 핵심적인 기능이며, 이 기능만으로도 독립적인 가치를 제공한다.

**Independent Test**: 학번과 비밀번호를 입력하여 호출하면 해당 학생의 전체 이수 과목 목록과 GPA를 반환하는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 유효한 학번/비밀번호, **When** 성적 조회를 요청하면, **Then** 전체 이수 과목 목록, 성적, GPA, 취득학점 집계를 반환한다
2. **Given** 잘못된 비밀번호, **When** 성적 조회를 요청하면, **Then** 명확한 로그인 실패 오류를 반환한다
3. **Given** 유효한 계정이지만 포털이 점검 중, **When** 성적 조회를 요청하면, **Then** 포털 접속 불가 오류를 반환한다

---

### User Story 2 - 수강내역 조회 (Priority: P2)

개발자가 학생의 현재 또는 특정 학기의 수강 과목 목록을 가져온다. 과목명, 학점, 담당교수, 시간/장소 정보가 포함된다.

**Why this priority**: 수강 중인 과목 정보는 시간표 앱, 학사 도우미 등 다양한 서비스에 활용 가능하다.

**Independent Test**: 학번/비밀번호로 호출 시 현재 학기 수강 과목 목록과 총 학점을 반환하는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 유효한 계정, **When** 수강내역을 요청하면, **Then** 현재 학기 수강 과목 목록과 총 학점을 반환한다
2. **Given** 유효한 계정과 특정 학기 지정, **When** 수강내역을 요청하면, **Then** 해당 학기의 수강 과목 목록을 반환한다

---

### User Story 3 - 장학이력 조회 (Priority: P3)

개발자가 학생이 수혜받은 장학금 이력을 가져온다. 장학금명, 금액, 학기 정보가 포함된다.

**Why this priority**: 장학 이력은 재정 관련 서비스나 학생 포트폴리오에 활용 가능하나, 성적/수강 대비 활용 빈도가 낮다.

**Independent Test**: 학번/비밀번호로 호출 시 장학금 수혜 이력과 총 금액을 반환하는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 유효한 계정, **When** 장학이력을 요청하면, **Then** 학기별 장학금명, 금액 목록과 총 금액을 반환한다

---

### User Story 4 - REST API 서버 (Priority: P2)

개발자가 HTTP API를 통해 외부 시스템(모바일 앱, 웹 프론트엔드 등)에서 위의 모든 기능을 호출할 수 있다.

**Why this priority**: 라이브러리 직접 사용 외에 REST API로도 제공해야 다양한 클라이언트에서 활용 가능하다.

**Independent Test**: REST API 엔드포인트에 학번/비밀번호를 POST로 전송하면 JSON 응답을 반환하는지 확인한다.

**Acceptance Scenarios**:

1. **Given** REST API 서버가 실행 중, **When** 유효한 credentials로 성적 조회 API를 호출하면, **Then** 성적 데이터를 JSON으로 반환한다
2. **Given** REST API 서버가 실행 중, **When** 잘못된 credentials로 호출하면, **Then** 401 상태 코드와 오류 메시지를 반환한다
3. **Given** REST API 서버가 실행 중, **When** 포털 접속 불가 시 호출하면, **Then** 502 상태 코드를 반환한다

---

### Edge Cases

- 학생이 재학 중이 아닌 경우 (휴학, 졸업, 제적 등) 조회 가능한 데이터 범위가 달라질 수 있다
- 포털이 비밀번호 변경을 요구하는 경우 (계정 잠금 상태)
- 짧은 시간 내 다수 요청 시 포털의 로그인 횟수 제한에 걸릴 수 있다
- 포털 점검 시간대에 접속 불가
- SSL/TLS 호환성 문제 (포털 서버의 오래된 TLS 설정)

## Requirements

### Functional Requirements

- **FR-001**: 시스템은 세종대 포털 SSO를 통해 학번/비밀번호로 인증할 수 있어야 한다
- **FR-002**: 시스템은 인증된 학생의 전체 기이수 성적 목록을 조회할 수 있어야 한다 (과목명, 학점, 등급, 평점, 이수구분, 재수강 여부)
- **FR-003**: 시스템은 성적 집계 정보를 제공해야 한다 (전체 GPA, 교양 GPA, 전공 GPA, 취득학점, 신청학점, 백분위)
- **FR-004**: 시스템은 특정 학기 또는 전체 학기의 수강내역을 조회할 수 있어야 한다
- **FR-005**: 시스템은 장학금 수혜 이력을 조회할 수 있어야 한다 (장학금명, 금액, 학기)
- **FR-006**: 시스템은 npm 패키지로 설치하여 코드에서 직접 호출할 수 있어야 한다
- **FR-007**: 시스템은 REST API 서버로 실행하여 HTTP를 통해 모든 기능에 접근할 수 있어야 한다
- **FR-008**: 시스템은 로그인 실패, 포털 접속 불가, 데이터 파싱 오류 등 오류 상황을 명확히 구분하여 알려야 한다
- **FR-009**: 시스템은 브라우저나 헤드리스 브라우저 없이 순수 HTTP 요청만으로 동작해야 한다
- **FR-010**: 시스템은 세종대 포털의 SSL/TLS 호환성 이슈를 자동으로 처리해야 한다

### Key Entities

- **GradeReport**: 학생의 전체 성적 보고서 — 학생 정보, 과목별 성적 목록, 학점 집계, GPA 포함
- **Grade**: 개별 과목 성적 — 년도, 학기, 과목명, 학수번호, 학점, 등급, 평점, 이수구분, 재수강 여부
- **EnrollmentReport**: 학기별 수강내역 — 수강 과목 목록, 총 학점
- **Enrollment**: 개별 수강 과목 — 과목명, 학점, 교수명, 시간/장소
- **ScholarshipReport**: 장학금 이력 — 장학금 목록, 총 금액
- **Scholarship**: 개별 장학금 — 년도, 학기, 장학금명, 금액

## Success Criteria

### Measurable Outcomes

- **SC-001**: 유효한 계정으로 성적 조회 시 5초 이내에 전체 성적 데이터를 반환한다
- **SC-002**: 유효한 계정으로 수강내역 조회 시 5초 이내에 해당 학기 수강 목록을 반환한다
- **SC-003**: 유효한 계정으로 장학이력 조회 시 5초 이내에 전체 장학 이력을 반환한다
- **SC-004**: 잘못된 비밀번호 입력 시 2초 이내에 명확한 로그인 실패 오류를 반환한다
- **SC-005**: 라이브러리를 npm install로 설치한 후 3줄 이내의 코드로 성적 데이터를 가져올 수 있다
- **SC-006**: REST API의 모든 엔드포인트가 자동 생성된 문서를 통해 확인 가능하다

## Assumptions

- 세종대 포털의 SSO 로그인 및 WebSquare JSON API 구조는 Python 프로토타입에서 검증된 것과 동일하게 유지된다
- 포털은 동일 사용자의 동시 프로그램 사용을 거부하므로, 각 데이터 조회는 독립된 HTTP 세션에서 수행해야 한다
- 포털의 SSL/TLS 설정이 오래되어 기본 설정으로는 접속 불가할 수 있으며 TLS 보안 수준 조정이 필요하다
- 포털은 로그인 횟수를 제한하며, 짧은 시간 내 다수 로그인 시 계정이 잠길 수 있다
- 사용자는 Node.js 18+ 환경에서 이 시스템을 사용한다
