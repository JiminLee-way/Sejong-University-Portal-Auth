## User Scenarios & Testing

### User Story 1 - 시간표 조회 (Priority: P1)

개발자가 학생의 현재 학기 시간표를 요일/교시별로 가져온다. 각 교시의 과목명, 교수명, 강의실 정보가 포함되어 시간표 앱이나 일정 관리 서비스에서 바로 활용할 수 있다.

**Why this priority**: 시간표는 학생들이 매일 참조하는 가장 기본적인 정보이며, 이미 구현된 수강내역 데이터를 기반으로 요일/교시 형태로 가공하여 제공하므로 활용도가 매우 높다.

**Independent Test**: 학번/비밀번호로 시간표 조회를 호출하면 현재 학기의 요일별 교시 배열을 반환하는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 유효한 계정, **When** 시간표 조회를 요청하면, **Then** 현재 학기의 요일/교시별 과목명, 교수명, 강의실 정보를 반환한다
2. **Given** 유효한 계정과 특정 학기 지정, **When** 시간표 조회를 요청하면, **Then** 해당 학기의 시간표를 반환한다
3. **Given** 수강 과목이 없는 학기, **When** 시간표 조회를 요청하면, **Then** 빈 시간표를 반환한다

---

### User Story 2 - 등록금 납부내역 조회 (Priority: P1)

개발자가 학생의 학기별 등록금 고지 및 납부 상태를 가져온다. 등록금 총액, 장학금 감면액, 실납부액, 납부 상태 등이 포함된다.

**Why this priority**: 등록금 정보는 재정 관리의 핵심이며, 장학금 이력과 결합하면 학생의 전체 재정 현황을 파악할 수 있다. 납부 기한 안내 서비스 등에 즉시 활용 가능하다.

**Independent Test**: 학번/비밀번호로 등록금 조회를 호출하면 학기별 등록금 고지/납부 내역을 반환하는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 유효한 계정, **When** 등록금 납부내역을 요청하면, **Then** 학기별 등록금 총액, 장학금 감면액, 실납부액, 납부상태를 반환한다
2. **Given** 신입생 계정(등록금 내역이 1건), **When** 등록금 납부내역을 요청하면, **Then** 해당 1건의 내역을 반환한다
3. **Given** 휴학 중인 학생, **When** 등록금 납부내역을 요청하면, **Then** 기존 납부 이력만 반환한다

---

### User Story 3 - 졸업요건 충족도 조회 (Priority: P2)

개발자가 학생의 졸업 요건 대비 현재 이수 현황을 가져온다. 졸업에 필요한 총 학점, 교양/전공 필수 이수 학점, 현재까지 이수한 학점, 부족 학점 등을 확인할 수 있다.

**Why this priority**: 졸업 요건 확인은 고학년 학생에게 매우 중요한 기능이며, 이미 성적 데이터의 creditSummary와 결합하면 더 풍부한 정보를 제공할 수 있다.

**Independent Test**: 학번/비밀번호로 졸업요건 조회를 호출하면 영역별 필요/이수/부족 학점 현황을 반환하는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 유효한 계정, **When** 졸업요건 충족도를 요청하면, **Then** 영역별(교양필수, 전공필수, 전공선택, 일반선택 등) 필요학점, 이수학점, 부족학점을 반환한다
2. **Given** 졸업 요건을 모두 충족한 학생, **When** 졸업요건 충족도를 요청하면, **Then** 모든 영역의 부족학점이 0이며 충족 상태를 반환한다
3. **Given** 복수전공 학생, **When** 졸업요건 충족도를 요청하면, **Then** 주전공과 복수전공 요건을 각각 표시한다

### User Story 4 - 도서 대출내역 조회 (Priority: P2)

개발자가 학생의 현재 도서관 대출 현황을 가져온다. 대출 중인 도서명, 대출일, 반납예정일, 연체 여부 등이 포함된다.

**Why this priority**: 도서관 기능은 이미 좌석 조회가 구현되어 있어 인프라를 재사용할 수 있으며, 연체 알림 서비스 등에 바로 활용 가능하다.

**Independent Test**: 학번/비밀번호로 도서 대출내역을 호출하면 현재 대출 중인 도서 목록을 반환하는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 유효한 계정이고 대출 중인 도서가 있을 때, **When** 도서 대출내역을 요청하면, **Then** 도서명, 대출일, 반납예정일, 연체여부를 포함한 목록을 반환한다
2. **Given** 대출 중인 도서가 없을 때, **When** 도서 대출내역을 요청하면, **Then** 빈 목록을 반환한다
3. **Given** 연체 도서가 있을 때, **When** 도서 대출내역을 요청하면, **Then** 해당 도서의 연체 상태와 연체일수를 표시한다

---

### User Story 5 - 도서관 좌석 예약/연장/반납 (Priority: P2)

개발자가 프로그래밍 방식으로 도서관 좌석을 예약, 연장, 반납할 수 있다. 기존의 조회 기능에 더해 실제 액션을 수행하는 기능이다.

**Why this priority**: 기존 좌석 조회 인프라(libseat 토큰 등)가 이미 있어 확장이 용이하며, 자동 연장 봇 등 실용적인 서비스를 만들 수 있다.

**Independent Test**: 빈 좌석에 예약 요청을 보내면 좌석이 배정되고, 연장/반납 요청이 정상 처리되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 유효한 계정이고 빈 좌석이 있을 때, **When** 좌석 예약을 요청하면, **Then** 해당 좌석이 배정되고 배정 정보를 반환한다
2. **Given** 좌석이 배정된 상태에서, **When** 좌석 연장을 요청하면, **Then** 사용 시간이 연장되고 남은 시간을 반환한다
3. **Given** 좌석이 배정된 상태에서, **When** 좌석 반납을 요청하면, **Then** 좌석이 반납되고 반납 완료 상태를 반환한다
4. **Given** 이미 좌석이 배정된 상태에서, **When** 다른 좌석 예약을 요청하면, **Then** 기존 좌석이 있다는 오류를 반환한다
5. **Given** 연장 횟수 제한에 도달했을 때, **When** 좌석 연장을 요청하면, **Then** 연장 불가 오류를 반환한다

---

### User Story 6 - 학사일정 조회 (Priority: P3)

개발자가 세종대의 학사일정(수강신청, 시험기간, 방학, 개강일 등)을 가져온다. 월별 또는 전체 학기의 주요 일정을 확인할 수 있다.

**Why this priority**: 학사일정은 공개 정보에 가깝지만, 포털 인증 후 조회 가능한 상세 일정을 제공하면 일정 알림 서비스 등에 유용하다.

**Independent Test**: 학번/비밀번호로 학사일정을 호출하면 학기 내 주요 일정 목록을 반환하는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 유효한 계정, **When** 학사일정 조회를 요청하면, **Then** 현재 학기의 주요 일정(일정명, 시작일, 종료일)을 반환한다
2. **Given** 유효한 계정과 특정 연도/학기 지정, **When** 학사일정 조회를 요청하면, **Then** 해당 학기의 학사일정을 반환한다

---

### User Story 7 - 공지사항 조회 (Priority: P3)

개발자가 학교 또는 학과의 공지사항 목록을 가져온다. 제목, 작성일, 카테고리 등을 포함하며 최신 공지를 빠르게 확인할 수 있다.

**Why this priority**: 공지사항은 학생 서비스의 보조 기능으로, 알림 봇이나 대시보드에 활용 가능하나 핵심 학사 데이터보다 우선순위가 낮다.

**Independent Test**: 학번/비밀번호로 공지사항을 호출하면 최근 공지 목록을 반환하는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 유효한 계정, **When** 공지사항 조회를 요청하면, **Then** 최근 공지사항 목록(제목, 작성일, 카테고리)을 반환한다
2. **Given** 유효한 계정과 카테고리 필터, **When** 공지사항 조회를 요청하면, **Then** 해당 카테고리의 공지만 반환한다
3. **Given** 유효한 계정과 페이지 지정, **When** 공지사항 조회를 요청하면, **Then** 해당 페이지의 공지 목록을 반환한다
4. **Given** 유효한 계정과 특정 공지 ID, **When** 공지사항 상세 조회를 요청하면, **Then** 해당 공지의 본문 내용을 반환한다

---

### Edge Cases

- 포털 내부 API 구조가 변경되어 기존 파싱 로직이 실패하는 경우
- 졸업요건 데이터가 학과/입학년도별로 상이한 경우
- 도서관 시스템(libseat)이 독립적으로 점검 중인 경우 (포털은 정상이지만 도서관만 접속 불가)
- 좌석 예약 시 동시 요청으로 인한 충돌 (다른 사용자가 동시에 같은 좌석 예약)
- 도서 대출내역이 학기 전환 시점에 갱신 지연되는 경우
- 공지사항의 특수문자/HTML 태그가 포함된 제목 처리
- 학사일정이 학기 중간에 변경/추가되는 경우

## Requirements

### Functional Requirements

- **FR-001**: 시스템은 학생의 현재 학기 시간표를 과목별 플랫 리스트(과목명, 교수명, 강의실, 요일, 교시)로 조회할 수 있어야 하며, 요일×교시 그리드 매트릭스로 변환하는 유틸리티도 제공해야 한다
- **FR-002**: 시스템은 특정 학기를 지정하여 해당 학기의 시간표를 조회할 수 있어야 한다
- **FR-003**: 시스템은 학기별 등록금 고지 내역을 조회할 수 있어야 한다 (등록금 총액, 장학금 감면액, 실납부액, 납부상태)
- **FR-004**: 시스템은 졸업 요건 대비 현재 이수 현황을 조회할 수 있어야 한다 (영역별 필요학점, 이수학점, 부족학점)
- **FR-005**: 시스템은 현재 도서관 대출 중인 도서 목록을 조회할 수 있어야 한다 (도서명, 대출일, 반납예정일, 연체여부)
- **FR-006**: 시스템은 도서관 좌석을 예약할 수 있어야 한다 (열람실 번호, 좌석 번호 지정)
- **FR-007**: 시스템은 배정된 좌석의 사용 시간을 연장할 수 있어야 한다
- **FR-008**: 시스템은 배정된 좌석을 반납할 수 있어야 한다
- **FR-009**: 시스템은 학사일정을 조회할 수 있어야 한다 (일정명, 시작일, 종료일)
- **FR-010**: 시스템은 학교/학과 공지사항 목록을 조회할 수 있어야 한다 (제목, 작성일, 카테고리)
- **FR-011**: 시스템은 공지사항을 카테고리별로 필터링하고 페이지 단위로 조회할 수 있어야 한다
- **FR-015**: 시스템은 개별 공지사항의 본문 내용을 조회할 수 있어야 한다
- **FR-012**: 모든 새 기능은 기존 SejongClient의 세션을 재사용하여 추가 로그인 없이 동작해야 한다
- **FR-013**: 모든 새 기능은 REST API 엔드포인트로도 제공되어야 한다
- **FR-014**: 좌석 예약/연장/반납 실패 시 명확한 오류 메시지를 반환해야 한다 (이미 배정됨, 연장 제한 초과, 빈 좌석 없음 등)

### Key Entities

- **Timetable**: 학기별 시간표 — 과목별 플랫 리스트(과목명, 교수명, 강의실, 요일, 교시)를 기본 반환하고, 요일×교시 그리드 매트릭스로 변환하는 유틸리티 함수도 제공
- **TuitionRecord**: 등록금 납부 내역 — 년도, 학기, 등록금 총액, 장학금 감면액, 실납부액, 납부상태
- **GraduationRequirement**: 졸업 요건 충족도 — 포털 API가 제공하는 구조를 그대로 반영 (학점 기반 영역별 요건 + 비학점 요건이 있으면 함께 포함)
- **BookLoan**: 도서 대출 내역 — 도서명, 대출일, 반납예정일, 연체여부, 연체일수
- **SeatAction**: 좌석 액션 결과 — 액션유형(예약/연장/반납), 좌석번호, 열람실명, 결과상태
- **AcademicEvent**: 학사일정 항목 — 일정명, 시작일, 종료일, 카테고리
- **Notice**: 공지사항 — 제목, 작성일, 카테고리, 작성자, 본문 내용 (상세 조회 시)

## Success Criteria

### Measurable Outcomes

- **SC-001**: 시간표 조회 시 5초 이내에 요일/교시별 정보를 반환한다
- **SC-002**: 등록금 납부내역 조회 시 5초 이내에 전체 학기 내역을 반환한다
- **SC-003**: 졸업요건 충족도 조회 시 5초 이내에 영역별 이수 현황을 반환한다
- **SC-004**: 도서 대출내역 조회 시 5초 이내에 현재 대출 목록을 반환한다
- **SC-005**: 좌석 예약/연장/반납 요청 시 3초 이내에 결과를 반환한다
- **SC-006**: 학사일정 조회 시 5초 이내에 일정 목록을 반환한다
- **SC-007**: 공지사항 조회 시 5초 이내에 목록을 반환한다
- **SC-008**: 모든 새 기능이 기존 세션을 재사용하여 추가 로그인 없이 동작한다
- **SC-009**: 모든 새 기능에 대해 라이브러리 호출과 REST API 엔드포인트가 모두 제공된다

## Assumptions

- 시간표 데이터는 기존 수강내역(enrollments) API의 시간/장소 필드를 파싱하여 요일/교시 형태로 변환할 수 있다
- 등록금 납부내역은 세종대 포털의 학부생학사정보(SELF_STUD) 메뉴에서 접근 가능한 WebSquare API로 조회 가능하다
- 졸업요건 충족도 데이터는 포털의 별도 PGM 키를 통해 조회 가능하다. 학점 기반 요건 외 비학점 요건(외국어인증, 졸업논문 등)은 포털 API가 제공하는 범위에 따라 포함 여부가 결정된다
- 도서 대출내역의 데이터 소스는 구현 시 API 리서치로 확정한다. 기존 libseat 시스템에서 대출 정보를 제공할 수 있으면 재사용하고, 별도 도서관 시스템(lib.sejong.ac.kr)이 필요한 경우 새로운 인증 흐름을 추가한다
- 좌석 예약/연장/반납은 기존 libseat 토큰 인증을 재사용하여 POST 요청으로 수행 가능하다
- 학사일정과 공지사항은 포털의 WebSquare API를 통해 JSON 형태로 조회 가능하다
- 포털의 동시 세션 제한 정책은 기존과 동일하며, 기존 세션 관리 방식으로 대응 가능하다
